
# terraces

**terraces** provides disaggregation algorithms for **terra** rasters that give 
**A**ggregation-**C**onsistent **E**stimators across **S**cales.

Bilinear and cubic interpolation are widely used for raster disaggregation, via 
`terra::disagg(method = "bilinear")` and similar functions in other packages. 
But when your raster values conceptually represent averages of finer-scale variation 
within each cell, which is perhaps the most common case, these interpolation 
methods are inappropriate. Their use for disaggregation assumes that raster values 
represent point samples at grid cell centers; wen used for disaggregating mean-based 
rasters, they produce fine-scale results that are not consistent with the observed
data and cannot be re-aggreagted to recover your original coarse values.

`terraces` provides *aggregation-consistent* disaggregation methods -- 
methods whose output, when re-aggregated, recovers the input exactly. Options
include bilinear and cubic variants of "presharpening" disaggreagation, 
and an implementation of pycnophylactic interpolation (Tobler, 1979). 
These methods differ mainly in their assumptions about within-block structure.

## When (not) to use `terraces`

**Use `terraces`** when cell values represent averages over the area of the 
cell. This is the typical case for most modern raster products:

- Coarsened DEMs (LiDAR aggregated to coarser resolution)
- Climate model output and reanalysis products (ERA5, CHELSA, WorldClim)
- Remote sensing products (radiance integrated over the pixel footprint)
- Gridded population counts and density rasters
- Any raster produced by `terra::aggregate(fun = "mean")` or equivalent

**Use standard bilinear disaggreagation** when cell values represent point 
measurements at cell centers, with no implied spatial extent:

- Surfaces interpolated from sparse point data (kriging, IDW, splines)
- Point-based model output at grid cell centers
- Synthetic rasters generated by evaluating a continuous function at grid points
- Coarsened rasters where the coarsening was nearest-neighbor sampling


## What's in the package
 
Three aggregation-consistent disaggregation methods:
 
- **`disagg_bl()`** — bilinear-style smoothness with a small precomputed
  prefilter applied to the coarse raster. Non-iterative; ~10% overhead
  over plain `terra::disagg()` at typical raster sizes.
- **`disagg_cub()`** — Keys cubic convolution (a = -0.5) with a 5×5
  prefilter kernel. Smoother than bilinear (C¹ continuity at coarse
  cell boundaries) and can place within-block extrema in cell interiors.
  May overshoot near sharp gradients due to cubic's negative side lobes.
- **`disagg_pyc()`** — Tobler's (1979) pycnophylactic interpolation,
  adapted to regular raster grids. Iteratively smooths the fine raster
  while restoring each coarse block's mean. Produces Laplacian-smooth
  output (no faceting at coarse cell boundaries). Iterative, slower
  (~10-50×), but smoother and more stable near sharp gradients.
  
```r
library(terra)
library(terraces)
 
coarse <- rast(nrows = 100, ncols = 100, vals = runif(10000))
 
# standard bilinear -- treats coarse values as point samples
fine_std <- disagg(coarse, fact = 5, method = "bilinear")
 
# aggregation-consistent alternatives:
fine_bl  <- disagg_bl(coarse, fact = 5)
fine_cub <- disagg_cub(coarse, fact = 5)
fine_pyc <- disagg_pyc(coarse, fact = 5)
```
 
## How they compare
 
All three methods are aggregation-consistent and substantially more
accurate than standard bilinear on typical aggregated rasters. They
differ in their assumptions about within-block structure:
 
| Property | `disagg_bl` | `disagg_cub` | `disagg_pyc` |
|----------|-------------|--------------|--------------|
| Within-block surface | Piecewise bilinear | Piecewise bicubic | Laplacian-smooth |
| Smoothness at cell boundaries | C⁰ | C¹ | C¹ |
| Extrema in block interiors | Vertices only | Yes | Yes |
| Can overshoot input range | No | Yes (small) | No |
| Iterative | No | No | Yes |
| Cost vs `terra::disagg(method="bilinear")` | ~1.1× | ~2× | ~10-50× |
 
Use `disagg_bl` when you want a fast, drop-in replacement for
`terra::disagg(method = "bilinear")` that's aggregation-consistent.
Use `disagg_cub` for smoother output with C¹ continuity at cell
boundaries, when bilinear's piecewise structure is visually objectionable
but you want a fast non-iterative method.
Use `disagg_pyc` when you want maximum smoothness, when sharp
coarse-scale gradients risk ringing (which `disagg_cub` may amplify), or
when guaranteed non-overshoot is required for physically-bounded data.

 
## How the prefilter method works
 
Standard bilinear interpolation, treated as a linear operator
"interpolate then aggregate," acts as a small smoothing convolution on
the coarse grid. The package derives this smoothing kernel analytically
(a 3×3 stencil), inverts it by direct linear solve, and applies the
inverse as a single focal pass on the coarse raster before standard
`terra::disagg()`. The inverse kernel depends only on the disagg factor;
it's computed once per factor and cached for the session.
 
This is structurally similar to prefilter-based B-spline interpolation
(Unser 1999), specialized to the case where the discrete data are areal
means rather than point samples.

## How the pycnophylactic method works

Standard Tobler (1979) iteration on a regular grid: start from a
nearest-neighbor or bilinear disaggregation, repeatedly smooth the fine
raster with a 3×3 Laplacian kernel, then add a per-block constant to
restore each coarse block's mean. Iterate until convergence (typically
20-100 iterations).

## References
